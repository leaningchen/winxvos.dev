# WinixOS 代码完善计划

## Context

WinixOS 是一个 64 位 x86 内核项目，目前已实现 VESA 帧缓冲文字输出、libc 基础库（stdio/string/ctype）、IDT/SMP 等核心功能。  
本次改进三个方向：
1. **补充中文注释**：部分头文件和 `.c` 文件缺少注释
2. **VESA 图形绘制**：`video.c` 目前只有文字输出，缺少图形 API（矩形、圆形、圆角矩形、多边形等）
3. **libc stdarg.h 改进**：参考 musl libc / Linux 内核做法，将 `va_list` 改为显式的 x86-64 ABI 结构体，`va_arg`/`va_end`/`va_copy` 手工实现；`va_start` 与 musl/Linux 相同，保留 `__builtin_va_start`（此为所有实现的共同做法，因为只有编译器能生成 prologue 中的寄存器保存代码）

### 任务 2：改写 stdarg.h

参考 musl libc (`arch/x86_64/bits/stdarg.h`) 和 Linux 内核的实际做法：

```
x86-64 System V ABI 变参调用约定：
- 整型/指针参数依次放入 rdi, rsi, rdx, rcx, r8, r9（最多6个）
- 超出6个的参数压栈（overflow_arg_area）
- va_start 由编译器在函数 prologue 中将上述寄存器保存到 reg_save_area
  （这是所有实现包括 musl/Linux/Minix 都保留 __builtin_va_start 的原因）
- va_arg 通过移动 gp_offset 指针逐个取出参数
```

**新 stdarg.h 结构**：

```c
/* x86-64 System V ABI 变参结构体（显式定义，非编译器内建类型）*/
typedef struct {
    unsigned int gp_offset;         /* 下一个通用寄存器参数在 reg_save_area 中的偏移
                                     * 取值范围: 0(rdi), 8(rsi), 16(rdx), 24(rcx),
                                     *           32(r8), 40(r9), 48=溢出到栈 */
    unsigned int fp_offset;         /* 浮点寄存器参数偏移（内核不用浮点，固定48）*/
    void        *overflow_arg_area; /* 栈上溢出参数区域指针（第7个及以上参数）*/
    void        *reg_save_area;     /* 寄存器保存区基地址（由编译器 prologue 填充）*/
} __va_list_tag;

typedef __va_list_tag va_list[1];   /* 数组类型，使其在传参时自动退化为指针 */

/* va_start: 必须由编译器填充 reg_save_area 和 overflow_arg_area
 * 与 musl libc / Linux 内核相同，此处保留 __builtin_va_start */
#define va_start(ap, last)   __builtin_va_start(ap, last)

/* va_end: 将 overflow_arg_area 清零，防止悬空使用 */
#define va_end(ap)           ((ap)[0].overflow_arg_area = (void *)0)

/* va_copy: 直接结构体赋值（与 musl 相同）*/
#define va_copy(dest, src)   ((dest)[0] = (src)[0])

/* va_arg: 手工实现，基于 x86-64 System V ABI 内存布局
 * 对于 sizeof(type) <= 8 的整型/指针类型：
 *   若 gp_offset < 48: 从 reg_save_area[gp_offset] 取值，gp_offset += 8
 *   否则: 从 overflow_arg_area 取值，移动指针（8字节对齐）
 * 内核中不使用浮点，故不处理 fp_offset 路径 */
#define va_arg(ap, type)  __va_arg_impl(ap, type)

/* 实现宏（避免多次求值 ap）*/
#define __va_arg_impl(ap, type)                                          \
    ( sizeof(type) <= 8                                                  \
      ? ( (ap)[0].gp_offset < 48u                                       \
          ? ( (ap)[0].gp_offset += 8u,                                  \
              *(type *)((char *)(ap)[0].reg_save_area +                  \
                        (ap)[0].gp_offset - 8u) )                       \
          : ( (ap)[0].overflow_arg_area =                               \
                  (char *)(ap)[0].overflow_arg_area +                   \
                  (((unsigned)(sizeof(type)) + 7u) & ~7u),              \
              *(type *)((char *)(ap)[0].overflow_arg_area -             \
                        (((unsigned)(sizeof(type)) + 7u) & ~7u)) ) )   \
      : *(type *)__builtin_va_arg_pack() )   /* 大类型回退到编译器 */
```

**注意**：Clang 编译时，只要使用 `-target x86_64-unknown-linux-gnu`（项目已配置），ABI 即为 System V，此方案完全适用。

### 任务 3：完善 VESA 图形驱动

#### 新增函数（video.h 声明 + video.c 实现）

```c
/* === 直线 === */
void video_draw_line(int x0, int y0, int x1, int y1, uint32_t color);
  // Bresenham 直线算法，支持任意方向，是多边形/矩形的基础

/* === 矩形 === */
void video_draw_rect(int x, int y, int w, int h, uint32_t color);
  // 画四条边（调用 video_draw_line）
void video_fill_rect(int x, int y, int w, int h, uint32_t color);
  // 双重循环填充像素（简单高效）

/* === 圆形 === */
void video_draw_circle(int cx, int cy, int r, uint32_t color);
  // Midpoint Circle（Bresenham）算法，8 次对称
void video_fill_circle(int cx, int cy, int r, uint32_t color);
  // Midpoint Circle 变体，每步画水平扫描线

/* === 圆角矩形 === */
void video_draw_round_rect(int x, int y, int w, int h, int r, uint32_t color);
  // 四条直线 + 四个 1/4 圆弧（Bresenham 圆弧，按象限分配到四角）
void video_fill_round_rect(int x, int y, int w, int h, int r, uint32_t color);
  // 中心区域用 fill_rect，上下弧形区域用圆弧扫描线

/* === 任意多边形 === */
void video_draw_polygon(const int *xs, const int *ys, int n, uint32_t color);
  // 依次连接顶点，首尾相连（调用 video_draw_line）
void video_fill_polygon(const int *xs, const int *ys, int n, uint32_t color);
  // 扫描线算法：遍历每行 y，求各边交点 x，排序后两两填充
  // 临时交点数组使用栈上固定大小（最多 VIDEO_MAX_POLY_VERTS=64 顶点）
```

#### 关键算法说明

**Bresenham 直线**：使用整数加减法，无浮点运算，适合内核环境。

**Midpoint Circle（8 次对称）**：  
从点 `(r, 0)` 开始，每步决策下一个像素，利用 `(±dx, ±dy)` 的 8 次对称性，一步画 8 个点（或 4 条水平线用于填充）。

**圆角矩形弧段**：将 Midpoint Circle 的 8 个对称点按象限分配到四个角落的圆心坐标，只取 1/4 圆弧段。

**扫描线多边形填充**：
```
for y = y_min to y_max:
    求所有边与 y 的交点 x 列表（整数，奇偶规则）
    对 x 列表排序
    两两配对：fill_rect(x[0]..x[1], y), fill_rect(x[2]..x[3], y)...
```

## 六、注意事项

- **stdarg ABI 依赖**：自定义 `va_arg` 依赖 `-target x86_64-unknown-linux-gnu` 的 System V ABI，项目 Makefile 已配置，风险可控
- **图形函数无堆内存**：所有绘制函数纯操作帧缓冲，不调用 kalloc
- **多边形交点数组**：使用栈上固定数组（`int xs[VIDEO_MAX_POLY_VERTS]`），避免动态分配；最大顶点数定义为 64
- **圆角矩形参数校验**：若 `r > w/2` 或 `r > h/2` 则自动夹紧，防止绘制错误
- **现有接口不变**：所有已有函数签名、行为不变
